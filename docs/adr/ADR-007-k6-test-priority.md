# ADR-007 k6 테스트 우선순위

## 상태와 결정일

Accepted. 결정일: 2026-07-09.

## 맥락과 문제

부하 관찰은 필요하지만 k6 결과만으로 포인트 잔액 정합성을 증명할 수는 없습니다. 먼저 해석 가능한 기준선을 만들지 않으면 Stress·Spike 결과도 비교하기 어렵습니다.

## 결정 동인

- 단계별 부하 결과를 비교 가능하게 만듭니다.
- 정합성 검증과 성능 관찰을 분리합니다.
- 도전 범위를 MVP와 혼동하지 않습니다.

## 검토한 선택지

| 선택지 | 판단 |
| --- | --- |
| Load → Stress → Spike | 채택. [테스트 전략](../testing/test-strategy.md)의 우선순위와 일치하며 기준선부터 확보합니다. |
| Spike부터 실행 | 제외. 기준선 없이 급증 결과를 해석하기 어렵습니다. |
| k6만으로 정합성 판정 | 제외. 포인트 정합성은 JUnit 동시성·DB 통합 검증이 필요합니다. |

## 결정과 이유

k6는 Load, Stress, Spike 순서로 사용하고 결과를 정합성 검증 근거로 사용하지 않습니다.

## 결과와 단점

부하 관찰의 목적과 정합성 검증의 목적이 분리됩니다. 반면 성능 수치만으로 락·트랜잭션의 정확성을 주장할 수 없고, 별도의 동시성 검증 evidence가 필요합니다.

## 재검토 조건

- 성능 목표나 테스트 대상 API가 확정될 때.
- 장시간 안정성 관찰이 필요한 근거가 생겨 Soak 후보를 실제 계획으로 승격할 때.
