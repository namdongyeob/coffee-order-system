# ADR-006 QueryDSL과 인덱싱

## 상태와 결정일

Planned. 결정일: 2026-07-09.

## 맥락과 문제

복잡한 검증 조회가 필요할 수 있으나 QueryDSL 도입이나 인덱스 추가가 자동으로 성능을 보장하지는 않습니다.

## 결정 동인

- 타입 안전한 조건 조합이 실제로 필요한지 확인합니다.
- 인덱스는 실행계획과 측정으로 판단합니다.
- 현재 MVP 범위를 근거 없이 넓히지 않습니다.

## 검토한 선택지

| 선택지 | 판단 |
| --- | --- |
| 필요한 조회에만 QueryDSL과 EXPLAIN 기반 인덱스 | 계획. [범위](../product/scope.md)의 도전 후보와 일치합니다. |
| 모든 조회를 QueryDSL로 선제 전환 | 제외. 단순 조회까지 복잡도를 늘릴 근거가 없습니다. |
| EXPLAIN 없이 인덱스 선제 추가 | 제외. 쓰기 비용과 실제 효과를 판단할 수 없습니다. |

## 결정과 이유

필요한 DB 검증 조회는 QueryDSL로 작성하고, 인덱스는 생성 SQL과 EXPLAIN 결과를 기준으로 결정합니다.

## 결과와 단점

성능 결정을 측정 가능한 근거에 연결할 수 있습니다. 반면 QueryDSL 의존성과 실행계획 evidence가 추가되며, 아직 구현·측정된 결과는 아닙니다.

## 재검토 조건

- 실제 조회 요구가 단순 JPA query method의 표현력을 넘을 때.
- EXPLAIN에서 병목과 인덱스 효과가 확인될 때.
