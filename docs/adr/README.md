# ADR 운영 규칙

## 상태 표기

ADR은 `Proposed`, `Accepted`, `Rejected`, `Deprecated`, `Superseded` 또는 프로젝트가 명시한 상태를 사용합니다. 상태와 결정일은 각 ADR에 기록합니다.

## 기존 결정 대체

기존 결정을 조용히 덮어쓰지 않습니다. 결정이 바뀌면 새 ADR을 만들고 새 문서에서 대체 대상 ADR을, 기존 문서에서 새 ADR을 서로 링크합니다. 기존 문서는 `Superseded` 상태로 남겨 당시 결정과 변경 이유를 추적할 수 있게 합니다.

## 검증 표기

실제로 실행해 관찰한 검증은 `실제 근거`로, 아직 구현·실행하지 않은 항목은 `계획된 검증`으로 분리합니다. 계획을 통과 결과나 현재 동작 근거로 표현하지 않습니다.
