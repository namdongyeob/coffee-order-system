Execution mode reason: 애플리케이션 동작을 바꾸지 않는 제한된 AI 문서 구조 변경이며, 독립 Combined Verifier와 CI로 링크, 정본, 범위를 검증합니다.

Level 5 reason: 애플리케이션 런타임 변경이 없는 문서 정리 작업입니다.
Level 6 required: NO
Level 6 reason: API 계약과 실제 HTTP 동작을 변경하지 않습니다.

## 완료 기준

- Review Gate와 QA Gate의 책임, 금지 범위, 판정 기준이 분리됩니다.
- AI Slop 제거 기준은 Review Gate에 통합됩니다.
- 완료 주장과 검증 Level의 정본 링크가 PR template, Evidence Guide, 테스트 전략으로 정리됩니다.
- 완료된 구현 계획은 archive로 이동하고, 서브에이전트 문서는 역할별 입출력 계약만 유지합니다.
- 삭제 또는 이동된 파일의 링크와 참조가 남지 않습니다.
- 기본 evidence와 Level 0 verification log를 남깁니다.
