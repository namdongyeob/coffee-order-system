# 모델·도구 매핑과 실행 환경 (비정본 조건부 참조)

이 문서는 핵심 실행 계약이 아니며 역할·검증 규칙의 정본이 아닙니다. 도구가 서브에이전트별 모델 override를 제공할 때의 후보 매핑과 실행 환경 확인 절차만 담습니다. 역할과 쓰기 권한의 정본은 [오케스트레이션 정책](orchestration-policy.md), 검증 실행 소유권의 정본은 [테스트 전략](../testing/test-strategy.md)입니다.

## 모델 사용 기준

도구가 서브에이전트별 모델 override를 제공하는 경우 아래 능력·비용 등급을 기준으로 실제 제품 모델을 매핑합니다. 기능이 노출되지 않은 환경에서는 부모 모델을 상속하고 역할 규칙만 유지합니다. 특정 제품·모델명은 핵심 실행 계약에 두지 않고 이 문서에서만 관리합니다.

| 역할 | 능력·비용 등급 | 기준 |
| --- | --- | --- |
| Main Coordinator | 중간 조정 등급 | 의존성과 작업 큐를 관리하고 결과를 전달합니다. |
| 일반 Spring 구현 | 균형 구현 등급 | 비용과 구현 품질의 균형을 맞춥니다. |
| 파일 탐색, 문서 위치 조사 | 저비용 읽기 등급 | 빠른 읽기 작업에 사용합니다. |
| 복잡한 동시성·멱등성 검토 | 고정밀 추론 등급 | 경쟁 조건과 실패 순서를 분석합니다. |

모델을 낮췄다는 이유로 테스트와 리뷰를 줄이지 않으며, 강한 모델을 사용했다는 이유로 evidence를 생략하지 않습니다. 실제 도구·CLI별 제품 모델명과 reasoning effort 매핑, 그 갱신 주기는 이 문서에서만 관리하고 핵심 실행 계약에 복제하지 않습니다.

## 실행 환경 확인

데스크톱 앱이 사용하는 CLI와 셸 `PATH`의 CLI 버전이 다를 수 있습니다. 모델 또는 Skill이 동작하지 않을 때는 먼저 두 실행 파일의 `--version`을 비교하고, 오래된 CLI의 모델 거부를 프로젝트 설계 실패로 판단하지 않습니다. 검증 evidence에는 사용한 CLI 버전, 모델, reasoning effort를 함께 기록합니다.
