# 동시성 전략

## 문제

같은 사용자의 주문 요청이 동시에 들어오면 동일한 잔액을 읽고 중복 차감이 발생할 수 있습니다.

## 결정

Redisson과 DB 비관적 락을 함께 사용합니다.

| 계층 | 목적 |
| --- | --- |
| Redisson | 여러 애플리케이션 인스턴스에서 같은 `userId` 주문/결제가 동시에 진입하는 것을 줄입니다. |
| DB 비관적 락 | `UserPoint` row의 최종 정합성을 보장합니다. |

## 테스트 시나리오

```text
초기 포인트: 10,000P
메뉴 가격: 4,000P
동시 주문 요청: 10개
기대 성공: 2개
기대 실패: 8개
최종 잔액: 2,000P
음수 잔액: 절대 없음
```
