# 도메인 규칙

## 메뉴

- 메뉴는 `id`, `name`, `price`를 가집니다.
- 메뉴 데이터는 고정 seed 데이터로 시작합니다.
- 메뉴 가격은 양수여야 합니다.

## 포인트

- 포인트 충전 금액은 1 이상이어야 합니다.
- 1회 충전 최대 금액 추천값은 1,000,000P입니다.
- 포인트 잔액은 음수가 될 수 없습니다.
- 충전 요청 시 `UserPoint` row가 없으면 생성합니다.
- 주문/결제 요청 시 `UserPoint` row가 없으면 실패합니다.

## 주문

- 주문/결제는 포인트로만 처리합니다.
- 주문 생성과 포인트 차감은 하나의 DB 트랜잭션으로 처리합니다.
- 주문에는 결제 금액 스냅샷인 `paidAmount`를 저장합니다.
- 최소 주문 상태는 `PAID`로 시작합니다.
- 검증 실패 시 결제 완료 주문을 생성하지 않습니다.

## 인기 메뉴

- 최근 7일 기준은 `orderedAt`이며 오늘을 포함합니다.
- Redis 랭킹은 파생 데이터입니다.
- DB 주문 데이터가 원천 데이터입니다.
- 동점 정렬 추천 기준은 주문 수 내림차순, 메뉴 ID 오름차순입니다.

## 이벤트

- 주문/결제가 성공하면 `OrderCompletedEvent`를 발행합니다.
- 이벤트에는 `eventId`, `orderId`, `userId`, `menuId`, `paidAmount`, `orderedAt`을 포함합니다.
- Consumer 멱등성은 `eventId` unique 처리 이력으로 보장합니다.
