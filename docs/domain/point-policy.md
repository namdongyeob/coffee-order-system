# 포인트 정책

| 정책 | 결정 |
| --- | --- |
| 환산 기준 | 1원은 1P입니다. |
| 최소 충전 금액 | 1P입니다. |
| 최대 충전 금액 | 1회 요청 기준 1,000,000P입니다. |
| 음수 잔액 | 허용하지 않습니다. |
| 충전 동시성 | DB 비관적 락으로 제어합니다. |
| 주문 동시성 | Redisson 진입 락과 DB 비관적 락을 함께 사용합니다. |
| 충전 시 포인트 row 없음 | 새로 생성합니다. |
| 주문 시 포인트 row 없음 | 실패 처리합니다. 현재 추천은 404입니다. |

## 최초 충전 동시성

동일한 `userId`에 최초 충전 요청이 동시에 들어와도 모든 유효 요청은 성공하고, 최종 잔액에는 모든 충전 금액이 한 번씩 반영되어야 합니다.
최초 row 생성 경쟁에서 MySQL unique 제약 충돌 또는 lock acquisition 실패가 발생하면 `REQUIRES_NEW` 트랜잭션에서 최대 3회 재조회·재시도합니다. 호출자에게 이미 트랜잭션이 있어도 각 시도는 독립된 새 트랜잭션을 사용합니다. 다른 요청이 생성한 row를 비관적 쓰기 락으로 다시 조회하므로 기존 row와 최초 생성 row 모두 같은 잠금 경계에서 잔액을 갱신합니다. 재시도 한도를 넘으면 마지막 DB 예외를 요청 실패로 반환합니다.
